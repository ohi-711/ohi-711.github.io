UML DIAGRAMS

CLASS DIAGRAMS

UML Diagrams how the structure of a program. Class diagrams show each class as a box with 3 sections: name, fields (name and type), and methods (name, type, parameters).

ASSOCIATIONS

UML diagrams also show connections between the classes that represent how the classes interact and depend on each other. It is indicated on a diagram through lines or arrows between two or more classes. There are several types of associations.

An association would mean that the two classes are related in some way. This is shown by a simple line with labels to describe the connection.

Composition is when one class is made up of at least one other class. In this situtation, the classes cannot exist independently. This is indicated on the diagram with a solid diamond on the line connecting the class, with the diamond being at the class that is composed of the other.

Aggregation is when one class is made up of at least one other class, but they can exist independently. This is shown with a hollow diamond, with the diamond being on the class which is made up of the other.

Generalization is when there is inheritance. This means that one class is derived from others and that the subclass has the same fields and methods of the original, as well as any new methods or fields. In a diagram, this is indicated through a hollow arrow, with the arrow pointing towards the more general class.
